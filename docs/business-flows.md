# 央企农业食材平台业务流程图

## 1. 总体业务闭环

```mermaid
flowchart LR
  A["供应商提供商品"] --> B["平台上架销售"]
  B --> C["C端用户手机号注册"]
  C --> D["浏览商品并下单"]
  E["富力/企业采购电子券"] --> F["平台生成券批次"]
  F --> G["实体卡发放给用户"]
  G --> H["用户绑定电子券"]
  H --> D
  D --> I["微信/支付宝/电子券支付"]
  I --> J["平台生成主订单"]
  J --> K["按供应商拆分子订单"]
  K --> L["供应商发货"]
  L --> M["用户收货/售后"]
  M --> N["平台与供应商结算"]
  E --> O["平台与采购方结算"]
```

## 2. 用户下单流程

```mermaid
flowchart TD
  A["用户进入小程序"] --> B["手机号登录"]
  B --> C["浏览商品"]
  C --> D["加入购物车"]
  D --> E["购物车按供应商分组"]
  E --> F["确认收货地址"]
  F --> G["选择电子券"]
  G --> H["选择微信/支付宝支付"]
  H --> I["提交订单"]
  I --> J{"支付是否成功"}
  J -- "否" --> K["订单待支付/关闭"]
  J -- "是" --> L["生成主订单"]
  L --> M["按供应商生成子订单"]
  M --> N["供应商分别发货"]
  N --> O["用户查看物流"]
  O --> P["确认收货或申请售后"]
```

## 3. 电子券采购、制卡、绑定、核销流程

```mermaid
flowchart TD
  A["富力/企业提出购券需求"] --> B["平台创建券批次"]
  B --> C["配置面额、数量、有效期、使用范围"]
  C --> D["生成券号和密码"]
  D --> E["导出制卡文件"]
  E --> F["制作实体卡"]
  F --> G["采购方发放给用户"]
  G --> H["用户输入券号密码或扫码绑定"]
  H --> I{"券是否有效"}
  I -- "否" --> J["提示无效/过期/已绑定"]
  I -- "是" --> K["绑定到用户账户"]
  K --> L["订单确认页选择电子券"]
  L --> M["支付时核销"]
  M --> N["生成核销记录"]
  N --> O["进入采购方对账报表"]
```

## 5. 售后退款流程

```mermaid
flowchart TD
  A["用户在订单详情选择子订单"] --> B["发起售后申请"]
  B --> C["选择原因并上传凭证"]
  C --> D["平台/供应商审核"]
  D --> E{"审核结果"}
  E -- "驳回" --> F["售后关闭"]
  E -- "通过" --> G["供应商处理退款/换货/补发"]
  G --> H{"是否退款"}
  H -- "否" --> I["换货/补发完成"]
  H -- "是" --> J["平台执行退款"]
  J --> K["微信/支付宝部分原路退回"]
  J --> L["电子券部分按确认规则处理"]
  K --> M["售后完成"]
  L --> M
```

## 6. 供应商结算流程

```mermaid
flowchart TD
```


## 8. 发票流关系

```mermaid
flowchart TD
  A["用户按订单申请发票"] --> B{"开票主体待确认"}
  B -- "平台统一开票" --> C["平台开具订单发票"]
  B -- "供应商分别开票" --> D["按子订单供应商开票"]
```

## 9. 需甲方确认的流程节点

- 电子券是否可以和微信/支付宝组合支付。
- 电子券是否找零或保留余额。
- 多张电子券是否可叠加。
- 退款时电子券是否恢复。
- 平台和供应商谁拥有最终退款审批权。
- 发票主体由平台还是供应商承担。
- 供应商结算周期和扣减规则。
