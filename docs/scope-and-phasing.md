# 央企农业食材平台功能范围与分期建议

## 1. 分期原则

一期优先跑通交易主链路，确保 C 端用户能购买、电子券能核销、供应商能发货、平台能对账结算。

后续版本再增加钱包、房产认证、复杂营销、数据大屏、会员体系等增强能力。

## 2. 一期 MVP 范围

### 2.1 C 端小程序

| 模块 | 功能 | 优先级 | 说明 |
| --- | --- | --- | --- |
| 登录注册 | 手机号登录、验证码 | P0 | 一期不强制房产认证 |
| 首页 | Banner、搜索、分类、电子券入口、推荐商品 | P0 | 突出央企/国企食材和电子券入口 |
| 商品 | 分类、列表、详情、供应商信息、库存、规格 | P0 | 商品需绑定供应商 |
| 购物车 | 加购、改数量、删除、按供应商分组 | P0 | 为拆单做前端表达 |
| 订单确认 | 地址、商品清单、电子券、支付方式、金额明细 | P0 | 支持组合支付需确认 |
| 支付 | 微信支付、支付宝支付、电子券支付 | P0 | 钱包后置 |
| 电子券 | 券号密码绑定、扫码绑定、我的券、使用记录 | P0 | 核心商业功能 |
| 订单 | 订单列表、订单详情、物流、确认收货 | P0 | 主订单 + 子订单展示 |
| 售后 | 售后申请、凭证上传、进度查看 | P1 | 按供应商子订单发起 |
| 我的 | 地址、发票抬头、客服、意见反馈 | P1 | 基础服务能力 |

## 3. 二期建议范围

| 模块 | 功能 | 价值 |
| --- | --- | --- |
| 钱包 | 充值、余额、消费明细、提现申请、人工审核 | 支持预充值场景 |
| 业主认证 | 房产信息、物业费场景核验、业主权益 | 支持富力社区专属场景 |
| 电子券增强 | 多券叠加、券包、品类券、限商品券、批量发放 | 支持更复杂企业福利 |
| 营销活动 | 满减、限时购、组合套餐、优惠专区 | 提升交易转化 |
| 供应链接口 | 供应商库存、物流、订单状态接口 | 降低人工操作 |
| 数据大屏 | 销售、券核销、供应商履约、区域数据 | 管理层展示 |
| 发票增强 | 电子发票接口、自动开票、开票失败处理 | 降低财务工作量 |

## 4. 暂不建议一期开发

| 功能 | 原因 |
| --- | --- |
| App | 小程序足够覆盖一期使用场景 |
| H5 商城 | 会增加适配和支付复杂度 |
| 复杂会员等级 | 对一期交易闭环不是必要条件 |
| 积分体系 | 可等交易数据稳定后再设计 |
| 团长/社区团购 | 与当前供应商直发模式不同 |
| 自动业主数据库对接 | 当前已确认一期不强制业主认证 |
| 复杂风控模型 | 一期可先做基础券码和订单风控 |

## 5. 验收标准建议

### 5.1 小程序验收

- 用户可通过手机号登录。
- 用户可浏览商品列表和商品详情。
- 用户可将不同供应商商品加入购物车。
- 购物车可按供应商分组展示。
- 用户可绑定电子券。
- 用户可提交订单并选择支付方式。
- 支付成功后可生成主订单和供应商子订单。
- 用户可查看订单详情和物流状态。
- 用户可针对子订单申请售后。

### 5.2 后台验收

- 平台可新增供应商。
- 平台可新增和上下架商品。
- 平台可查看主订单和子订单。
- 平台可创建电子券批次并生成券码。
- 平台可查看电子券绑定和核销记录。
- 平台可处理售后申请。
- 平台可生成供应商结算数据。
- 平台可导出订单、券、结算相关报表。

### 5.3 供应商后台验收

- 供应商只能查看自己的子订单。
- 供应商可录入物流并确认发货。
- 供应商可查看售后申请。
- 供应商可查看销售和结算数据。

## 6. 关键待确认项

| 问题 | 影响 |
| --- | --- |
| 电子券是否支持组合支付 | 影响订单确认页、支付接口、金额拆分 |
| 电子券是否找零 | 影响券余额模型和退款逻辑 |
| 是否允许多券叠加 | 影响券选择交互和核销规则 |
| 退款时券是否恢复 | 影响售后流程和财务对账 |
| 发票主体是谁 | 影响发票管理和订单详情 |
| 供应商结算周期 | 影响结算单生成规则 |
| 平台收入模式 | 影响结算金额计算 |
| 售后退款审批责任 | 影响平台和供应商后台权限 |
