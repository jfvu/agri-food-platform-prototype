# 央企农业食材平台前端设计方案

## 1. 产品定位修正

本项目前端按“食材电商交易平台 + 企业电子券核销 + 多供应商履约结算”设计。

平台不再以“封闭业主商城”作为一期核心前提。一期用户通过手机号注册即可购买商品，配送地址由用户自行填写。富力/企业采购实体电子券后发放给 C 端用户，用户在商城绑定券码并用于订单支付。供应商负责商品质量、发货、物流和售后履约，平台作为统一销售渠道、交易收款方和结算管理方。

## 2. 前端端别

### 2.1 C 端微信小程序

面向普通消费者，用于浏览商品、绑定电子券、下单支付、查看订单和售后。

核心目标：

- 快速购买食材
- 支持微信/支付宝/电子券支付
- 支持多供应商商品混合下单
- 让用户感知央企/国企供应商品质
- 降低实体卡券绑定和核销门槛

### 2.2 平台管理后台

面向平台运营、财务、卡券管理员和客服。

核心目标：

- 管商品、供应商、订单、电子券、财务结算
- 查看全平台交易和履约情况
- 处理售后、退款、对账和报表导出

### 2.3 供应商后台

面向北大荒、中粮、首农等供应商或其运营人员。

核心目标：

- 查看本供应商订单
- 录入物流并发货
- 处理售后
- 查看销售和结算数据

供应商后台可以一期并入平台后台，通过角色权限区分；如果供应商数量较多，再拆成独立入口。

## 3. C 端小程序信息架构

底部 Tab 建议采用 4 个入口：

```text
首页 / 分类 / 购物车 / 我的
```

### 3.1 首页

首页承担“信任建立 + 商品转化 + 卡券入口”的作用。

页面结构：

```text
顶部搜索栏
轮播 Banner
快捷入口：电子券绑定、订单、客服、企业福利专区
商品分类导航
央企品牌专区
推荐商品流
```

核心模块：

- 搜索栏：支持商品名、品牌、品类搜索。
- Banner：展示平台活动、企业福利券使用引导、重点品牌。
- 快捷入口：绑定电子券、我的订单、联系客服。
- 品牌专区：北大荒、中粮、首农等供应商入口。
- 商品推荐：按销量、活动、平台推荐展示。

设计注意：

- 首页不再强调“仅业主可用”，避免与实际注册逻辑冲突。
- 可以强调“央企/国企优选食材”“企业福利券可用”“供应商直发包邮”。

### 3.2 分类页

分类页用于高效筛选商品。

一级分类建议：

- 米面粮油
- 杂粮干货
- 生鲜食材
- 副食调味
- 福利礼盒
- 品牌专区

筛选项：

- 品牌/供应商
- 价格区间
- 销量
- 是否可用电子券
- 库存状态

商品卡片字段：

- 商品图
- 商品名称
- 供应商名称
- 价格
- 是否包邮
- 是否支持电子券
- 库存/售罄状态

### 3.3 商品详情页

商品详情页需要强化供应商责任和品质信任。

页面结构：

```text
商品图片
价格与商品名称
供应商信息
包邮与发货说明
电子券可用提示
规格选择
商品详情
资质/产地/执行标准
售后说明
底部操作栏：客服 / 购物车 / 加入购物车 / 立即购买
```

关键字段：

- 商品名称
- 商品图片
- 售价
- 供应商
- 品牌
- 产地
- 规格
- 保质期
- 执行标准
- 库存
- 发货说明
- 售后规则
- 发票说明

### 3.4 购物车

购物车必须按供应商分组，因为后续要拆单发货。

页面结构：

```text
供应商 A
  商品 1
  商品 2

供应商 B
  商品 3

底部结算栏：已选商品数量 / 合计金额 / 去结算
```

交互规则：

- 用户可以一次勾选多个供应商商品结算。
- 前端展示为一个购物车结算动作。
- 支付成功后，系统生成一个主订单和多个供应商子订单。
- 供应商包邮，因此购物车不展示复杂运费计算；仅在商品或结算页提示“供应商包邮”。

### 3.5 订单确认页

订单确认页是电子券核销和拆单提示的关键页面。

页面结构：

```text
收货地址
商品清单，按供应商分组
发票信息
电子券
支付方式
金额明细
提交订单
```

金额明细：

```text
商品总额
电子券抵扣
实付金额
```

支付方式：

- 微信支付
- 支付宝支付
- 电子券支付
- 电子券 + 微信/支付宝组合支付

建议规则：

- 允许电子券与微信/支付宝组合支付。
- 如果电子券不足以覆盖订单金额，剩余部分走第三方支付。
- 如果电子券超过订单金额，需根据业务另行确认是否保留余额、作废差额或禁止使用。

### 3.6 电子券模块

电子券是本项目的核心商业工具，入口要明显。

入口位置：

- 首页快捷入口
- 我的页
- 订单确认页

页面包括：

#### 3.6.1 绑定电子券

绑定方式：

- 输入券号 + 密码

绑定后展示：

- 券名称
- 券面额
- 有效期
- 可用范围
- 是否已过期
- 是否已核销

#### 3.6.2 我的电子券

Tab：

```text
可用 / 已使用 / 已过期
```

券卡片信息：

- 券面额
- 券名称
- 有效期
- 使用范围
- 来源企业/批次，后台可见，前端可选择隐藏
- 去使用按钮

#### 3.6.3 券使用记录

展示：

- 绑定时间
- 使用订单
- 抵扣金额
- 券状态

### 3.7 支付结果页

支付成功页要处理多供应商拆单后的用户预期。

页面内容：

```text

```

### 3.8 我的订单

订单列表可以按主订单展示，但详情页必须展示供应商子订单。

订单状态：

- 待支付
- 待发货
- 部分发货
- 待收货
- 已完成
- 售后中
- 已关闭

订单详情结构：

```text
主订单信息
支付信息
供应商 A 子订单
  商品
  物流
  售后入口
供应商 B 子订单
  商品
  物流
  售后入口
发票信息
```

### 3.9 售后申请

售后要按供应商子订单发起，而不是按整个主订单发起。

售后类型：

- 质量问题
- 错发
- 漏发
- 破损
- 物流异常

申请信息：

- 选择商品
- 售后原因
- 上传图片/视频凭证
- 说明
- 联系方式

退款去向：

- 第三方支付部分按原路退回。
- 电子券部分按规则恢复或不恢复，需要继续确认。

### 3.10 我的

页面结构：

```text
用户信息
我的订单
我的电子券
收货地址
发票抬头
客服中心
意见反馈
设置
```

一期不建议突出钱包入口。钱包、提现、余额明细可作为二期功能。

## 6. 核心交互流程

### 6.1 用户下单流程

```text
手机号登录
  ↓
浏览商品
  ↓
加入购物车
  ↓
购物车按供应商分组
  ↓
提交订单
  ↓
选择电子券和支付方式
  ↓
支付成功
  ↓
生成主订单 + 多个供应商子订单
  ↓
供应商分别发货
  ↓
用户分别查看物流/确认收货/申请售后
```

### 6.2 电子券绑定和使用流程

```text
企业/富力采购实体卡
  ↓
后台创建券批次并生成券码
  ↓
导出制卡文件
  ↓
实体卡发放给用户
  ↓
用户输入券号密码或扫码绑定
  ↓
订单确认页选择电子券
  ↓
支付时核销
  ↓
后台形成券核销记录和采购方对账数据
```

### 6.3 供应商履约流程

```text
用户支付成功
  ↓
平台拆分供应商子订单
  ↓
供应商后台接收订单
  ↓
供应商发货并录入物流
  ↓
用户查看物流并确认收货
  ↓
订单进入可结算范围
  ↓
平台按周期与供应商结算
```

## 9. 仍需确认后才能定稿的前端规则

以下规则会直接影响页面和交互，需要尽快确认：

1. 电子券是否可以和微信/支付宝组合支付。
2. 电子券是否支持一次用不完保留余额。
3. 一单是否允许使用多张电子券。
4. 电子券支付订单发生退款时，券是否恢复。
5. 过期券是否完全失效，还是售后退款时可特殊恢复。
6. 发票由平台统一开，还是供应商分别开。
7. 电子券支付金额是否允许给 C 端用户开票。
8. 供应商结算周期是订单完成后、T+N，还是月结。
9. 平台收入模式是佣金、服务费，还是供销差价。
10. 售后退款是供应商确认后退，还是平台先退后从结算款扣。

## 10. 设计优先级建议

一期前端优先做交易闭环：

```text
C 端购买
电子券绑定/核销
多供应商拆单
供应商发货
售后申请
平台结算
```

暂缓：

- 钱包充值和提现
- 业主房产认证
- 数据大屏
- 复杂营销活动
- 积分会员
- 团购预售
