# 央企农业食材平台产品需求说明书

## 1. 项目概述

本项目建设一个面向 C 端用户的农业食材电商平台，重点承载央企/国企优质食材销售、企业电子券核销、多供应商履约发货、平台统一收款与结算管理。

平台一期采用开放注册模式，用户通过手机号注册即可购买商品。富力或企业客户可向平台采购实体电子券，并发放给员工、业主或 C 端客户。用户在小程序内绑定电子券后，可用于平台商品消费。商品由供应商负责质量、发货、物流和售后，平台负责商品展示、交易收款、订单拆分、卡券核销、对账结算和运营管理。

## 2. 建设目标

- 搭建 C 端食材商城，支持用户浏览、下单、支付、查物流、申请售后。
- 支持企业/富力采购实体电子券，完成券码生成、绑定、核销、过期和统计。
- 支持多供应商商品混合下单，支付后按供应商自动拆分子订单。
- 支持供应商接单、发货、售后处理和结算数据查看。
- 支持平台统一管理商品、供应商、订单、卡券、售后、发票和财务报表。

## 3. 业务定位

平台核心定位为：

```text
农业食材电商销售渠道
+ 企业电子券消费核销平台
+ 多供应商履约管理平台
+ 平台统一收款与结算系统
```

一期不将“业主房产认证”作为强制业务前提，后续可根据物业费、社区服务或业主专属权益场景扩展业主认证能力。

## 4. 业务角色

### 4.1 C 端用户

通过微信小程序注册、浏览商品、绑定电子券、下单支付、查看订单、申请售后、申请发票。

### 4.2 平台运营人员

负责商品、首页、供应商、订单、售后、用户、公告等日常运营。

### 4.3 卡券管理员

负责电子券批次创建、券码生成、导出制卡、激活、冻结、作废、核销统计。

### 4.4 财务人员

负责企业购券结算、供应商销售结算、支付渠道对账、退款核对、发票管理和报表导出。

### 4.5 供应商

负责本供应商商品的质量、库存、发货、物流录入、售后协同和结算确认。

### 4.6 企业/富力采购方

向平台采购电子券，用于企业福利、客户福利、业主福利或其他营销发放场景。

## 5. 端别范围

### 5.1 微信小程序

面向 C 端用户，承担商品购买、电子券绑定和订单服务。

### 5.2 平台管理后台

面向平台内部人员，承担运营、商品、卡券、订单、售后、结算、报表等管理。

### 5.3 供应商后台

面向供应商，承担订单接收、发货、售后协同和结算查看。供应商后台一期可与平台后台共用入口，通过权限隔离数据。

## 6. 核心业务规则

### 6.1 用户注册规则

- 一期用户通过手机号注册登录。
- 不强制绑定房产信息。
- 一个用户可添加多个收货地址。
- 收货地址不限制为富力小区地址。

### 6.2 商品规则

- 商品由平台或供应商维护。
- 商品需绑定供应商。
- 商品需展示品牌、产地、规格、保质期、执行标准、售后说明等信息。
- 商品可配置是否支持电子券支付。
- 供应商负责商品质量、发货和物流。

### 6.3 支付规则

- 订单支持微信支付、支付宝支付、电子券支付。
- 建议支持电子券 + 微信/支付宝组合支付。
- 钱包充值和提现不作为一期核心功能。
- 如后期增加钱包，提现需后台人工审核。

### 6.4 电子券规则

- 电子券由平台后台批量生成。
- 电子券主要来源为富力或企业采购。
- 企业/富力采购后，可制作为实体卡并发放给用户。
- 用户通过券号密码或扫码方式绑定电子券。
- 电子券不等同现金。
- 电子券过期失效。
- 电子券可配置面额、有效期、使用范围、状态。

待确认规则：

- 是否允许一单使用多张电子券。
- 是否允许电子券一次用不完保留余额。
- 退款时电子券是否恢复。
- 过期券发生售后退款时是否特殊处理。

### 6.5 订单拆单规则

- 用户可在购物车中混合购买多个供应商商品。
- 用户支付时只感知一个主订单。
- 支付成功后，系统按供应商拆分为多个子订单。
- 每个供应商只处理属于自己的子订单。
- 用户可在订单详情中查看不同供应商子订单的物流和售后状态。

### 6.6 履约规则

- 商品由供应商负责发货。
- 供应商负责包邮。
- 供应商在后台录入物流公司和物流单号。
- 多供应商订单可能分开发货、分开签收、分开售后。

### 6.7 售后规则

- 售后按供应商子订单发起。
- 售后类型包括质量问题、错发、漏发、破损、物流异常等。
- 用户提交售后原因、凭证和说明。
- 供应商负责处理具体履约问题。
- 平台可保留退款审核和最终处理权。

### 6.8 发票规则

- 用户按订单申请发票。
- 当前确认方向为按订单开票。

待确认规则：

- 发票由平台统一开具，还是供应商分别开具。
- 电子券支付部分是否允许向 C 端用户开票。

### 6.9 结算规则

结算对象分为两类：

- 企业/富力采购电子券：谁采购电子券，平台就与谁结算。
- 供应商销售货款：供应商卖出多少商品，平台按结算周期与供应商结算。

待确认规则：

- 供应商结算周期。
- 平台佣金、服务费或价差规则。
- 售后退款是否从供应商结算款中扣减。

## 10. 非功能需求

- 小程序页面加载流畅，适配主流微信版本。
- 后台支持多角色权限控制。
- 关键操作需要操作日志。
- 支付、券码、订单、结算等核心数据需要可追溯。
- 订单、电子券、结算报表支持 Excel 导出。
- 个人信息、支付信息、券码信息需加密或脱敏展示。

## 11. 一期待确认事项

1. 电子券是否可与微信/支付宝组合支付。
2. 电子券是否允许一次用不完保留余额。
3. 一单是否允许使用多张电子券。
4. 退款时电子券是否恢复。
5. 发票由平台统一开，还是供应商分别开。
6. 电子券支付部分是否允许向用户开票。
7. 供应商结算周期。
8. 平台收入模式，佣金、服务费或供销差价。
9. 售后退款审批责任和扣款规则。
